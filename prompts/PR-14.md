# PR-14 — Import seed history retroactively + better UI surface

PR-1 shipped `SeedTraceIngestor`, which reads Seed's `*.seed-trace.json` sidecars and creates ledger events + Artifacts. But the ingestion currently only fires on **FileWatcher events** — when a sidecar is written or modified *while* Provenance is watching. This leaves a gap:

**The gap:** if a user transplants seeds → board ends up in a folder → user later connects that folder to Provenance → the sidecars exist on disk but Provenance silently ignores them.

This prompt closes that gap and improves how seed-derived history is surfaced.

## Scope

1. **Scan-on-connect** — when a project is connected (initial connect OR re-resolve after move), walk the folder for existing `*.seed-trace.json` files and ingest each one.
2. **Idempotent ingestion** — same sidecar processed twice doesn't create duplicate ledger events or duplicate Artifacts.
3. **Better UI surface** — make seed-derived Artifacts and ledger events visually distinct so the user can see "this idea was planted Tuesday, refined Wednesday, became this card Friday" at a glance.

## (1) Scan on project connect

In the project-connect flow (per PR-9 + PR-10), after the project is added to the store and its bookmark/sourceURL is resolved, scan the project folder:

```swift
func scanForSeedTraces(in projectRoot: URL) -> [URL] {
    var found: [URL] = []
    let enumerator = FileManager.default.enumerator(
        at: projectRoot,
        includingPropertiesForKeys: nil,
        options: [.skipsHiddenFiles, .skipsPackageDescendants]
    )
    while let url = enumerator?.nextObject() as? URL {
        if url.lastPathComponent.hasSuffix(".seed-trace.json") {
            found.append(url)
        }
    }
    return found
}
```

Skip `.ledger/` and `.git/` and any other hidden directories during the walk.

For each found sidecar, call `SeedTraceIngestor.ingest(_:)` (the same code path used by FileWatcher). The ingestor must be idempotent (see section 2).

This scan runs:
- Once on first project connect (initial)
- Once on every project re-resolve (PR-10's bookmark/Locate flow recovers a moved project)
- NOT on every project re-open (would be too aggressive — once per session is fine via a "seen sidecars" set in ProjectState)

## (2) Idempotency

The ingestor needs to avoid creating duplicate ledger events and duplicate Artifacts when the same sidecar is seen multiple times. Two approaches:

**A. Content-hash based.** For each seed entry in a sidecar, hash `(boardPath, originalGardenPath, promotedAt)`. Store seen hashes in a `.ledger/seed-traces-seen.json` index. Skip seeds whose hash is already there.

**B. Existence check.** Before creating a new Artifact for a seed entry, check whether an Artifact with matching `title: "Seed: <seed title>"` and matching `timestamp` already exists. Skip if so.

Either works. **A is more robust** for sidecars that get appended over time (Seed adds new seeds to an existing sidecar on subsequent transplants). Go with A.

## (3) "Seeds" surface in the Artifacts tab

Currently, seed-derived Artifacts appear interspersed with other Artifacts (ScannedNote, Image, etc.). Add visual + organizational treatment:

- **New Artifact subtype:** `.seedHistory` (extend the existing Artifact type enum). Seed-derived Artifacts use this type going forward; back-fill existing seed-derived ones if any (their title starts with "Seed: ").
- **Distinct icon for seed Artifacts:** SF Symbol `leaf` (or whatever the Seed app uses for its app icon glyph), 14pt, `Brand.textBrand` color.
- **Artifacts tab filter chip:** add a chip row at the top of the Artifacts tab (similar to the ledger filter from PR-2): All · Seeds · Notes · Images · Audio · Other. When "Seeds" is active, show only `.seedHistory` artifacts.
- **Click a seed Artifact** → opens a detail sheet (existing pattern from PR-1) that shows the full history timeline from the sidecar. Make this sheet readable as a story: "Planted Tue Apr 10 → Edited Wed Apr 11 → Merged with another seed Thu → Transplanted to *Act 1.sceneboard* Fri."

## (4) "Seeds" section on Home view (optional, ship if quick)

If time permits, add a "Recent Seeds" section to Provenance's Home view (above or beside the project cards strip):

- Shows the 5 most-recently-imported seed Artifacts across all connected projects
- Each row: small leaf glyph, seed title, "imported X ago", small project-name caption
- Click → navigates to that project's Artifacts tab, "Seeds" filter active, scrolled to that seed

If this adds more than ~1 hour, skip it. The Artifacts-tab surface (section 3) is the primary deliverable.

## Don't

- Don't change Contract A (`*.seed-trace.json` schema). The sidecar format is stable.
- Don't try to import history without a sidecar (no "guess from filename" heuristics). The sidecar is the source of truth.
- Don't write back to the sidecar from Provenance.
- Don't ship "Recent Seeds" on Home if it adds significant time. Artifacts-tab surface is enough.
