# PR-12 — Provenance polish: settings, new-project flow, Home declutter

Three related Provenance refinements bundled because they're each small. The user has flagged each as friction in their testing.

## (1) Move "Track pasted sources" toggle to Settings

PR-2 added a pasteboard observer with a toggle controlling whether paste events are recorded. PR-2's spec placed two controls:
- **Global toggle** in Settings → Privacy section ("Track paste sources")
- **Per-project override** on the Overview tab ("Track pastes in this project")

The user wants both consolidated into Settings. Specifically:
- Keep the global toggle in Settings → Privacy
- **Remove the per-project override from the Overview tab**
- If per-project overrides were genuinely useful, they can return as a Settings-window submenu later. For now: one global setting, off-by-default behavior per project, no Overview clutter.

Move any per-project state currently associated with the Overview toggle into a global setting (or just drop it if it wasn't widely used yet).

## (2) "+ New Project" button in the toolbar

Provenance currently relies on File → Open Project and the sidebar `+` button. Match the pattern used in Seed (new seed button) and Works (add works button) — put an obvious "+ New Project" affordance in the toolbar.

**Placement:** in the unified toolbar (per §5a), on the right side (alongside Export — which already lives in Project view's tab strip per PR-7, but a New Project action belongs at the app level, not project level).

**Style:**
- SF Symbol `plus.circle` (or `plus` alone) — 14pt, `Brand.textSecondary`, borderless
- Tooltip: "Connect Project…"
- Action: invokes the same Connect Project flow from PR-9 (accessory view + Create New Folder sub-sheet)
- Visible in both Home view and Project view (always available)

**Keep the sidebar `+`** — it still works for users who prefer it. The toolbar button is an additional, more visible entry point.

## (3) Home view declutter

PR-6 shipped the Home view with greeting, contextual prompt, check-in field, status picker, project cards strip, and "This week" aggregates. The user finds it too busy. Refinements:

### Center the prompt and text input

Currently, the prompt + check-in field is left-aligned (per design standards padding conventions). For Home view specifically, **center the prompt and field horizontally** in the available pane width, with a max-width of ~560pt.

The greeting line at top stays left-aligned at `Brand.spaceLG`. Only the prompt block (prompt text + check-in field + status picker + save button) is centered.

This gives the Home view a calmer, more focused feel — the user's attention is drawn to the one action that matters (writing the check-in).

### Project-aware prompt updates

The Home view currently selects a "qualifying project" once (per PR-6's rule) and shows the prompt for that project. The user wants the prompt to **update dynamically** when they click a different project card in the cards strip below.

Behavior:
- The cards strip shows all projects
- Clicking a card (gentle tap) now BOTH: highlights that card AND updates the prompt above to reference that project ("How did your last session on `<that project>` go?")
- The check-in field, when saved, goes to whatever project is currently named in the prompt
- A double-click (or some other "Open project" affordance) navigates into the project's tabs view (the old single-click behavior). OR: change to a separate "Open" button on each card.

If the dual-tap-vs-click distinction is awkward, an acceptable simpler design:
- Single-click on a project card SELECTS that project as the prompt context (no navigation)
- An "Open" affordance on the card (a small chevron or button in the corner) navigates to the project's tabs view

Pick the cleanest implementation; the goal is fluid switching between which project the prompt is for.

### Better placeholder text

The check-in field currently has placeholder "A sentence or two is plenty." (from PR-6). Replace with something that signals this is the check-in feature:

- "Ready to check in? Type a sentence or two."

Keep the placeholder short and inviting. Italic styling is acceptable.

### Remove "X days ago" from the sidebar

Each project row in the sidebar currently shows a "4 days, 22 hr" relative-time line under the project name (from earlier Provenance design). The user finds this oppressive — feels like a productivity tracker shaming them about gaps.

**Remove the relative-time line from sidebar project rows entirely.** The row becomes:

```
●  ProjectName
   /path/to/folder       (existing folder path subtitle)
```

If the user wants to know when they last worked on a project, the Overview tab and the Versions tab both surface it richly. The sidebar doesn't need to.

The status dot at the row's left edge stays (the small colored dot indicating the last check-in's status).

### What stays the same in Home

- Greeting (top, time-of-day aware) — stays as-is, left-aligned
- Welcome empty state (when no projects connected) — stays as-is
- "This week" aggregate counts — stays as-is, at the bottom
- The project cards strip itself (just the click behavior changes)

## Acceptance

### Settings
- Settings window has the global "Track paste sources" toggle.
- The Overview tab no longer has a per-project paste tracking toggle.
- Existing per-project overrides (if any users had set them) silently revert to the global setting.

### New Project button
- Provenance's toolbar shows a `plus.circle` icon on the right.
- Click → Connect Project flow opens (same as sidebar +).
- Available from both Home view and Project view.

### Home view
- Greeting at top, left-aligned, time-of-day aware.
- Prompt block (prompt text + field + status + save) is horizontally centered in the pane, max ~560pt wide.
- Project cards strip below the prompt block.
- Clicking a project card updates the prompt above to reference that project's display name.
- Check-in saved goes to the project named in the prompt.
- Placeholder text in the check-in field reads "Ready to check in? Type a sentence or two."
- Sidebar project rows show: status dot, project display name, folder path. **No "X days ago" line.**
- "This week" summary remains at the bottom.

## Don't

- Don't add new visual chrome to the Home view. The declutter is about removing, not adding.
- Don't auto-save the check-in on each keystroke. Save is explicit (button or ⌘↩).
- Don't show the per-project paste toggle anywhere — it's gone in v1.
- Don't add a "Settings" button to the Home view itself. Settings has its own window (⌘,).
