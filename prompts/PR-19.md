## (2) Home screen — typography

**Expected visual:** The word "Good night." (or whatever time-of-day greeting) renders in a serif italic, light weight, large (~34pt). If it currently renders in bold system UI, the font change did not land.

### 2a — Greeting: Palatino Italic light

```swift
// SwiftUI
Text(greetingString)
    .font(.custom("Palatino-Italic", size: 34))
    .fontWeight(.light)

// AppKit
textField.font = NSFont(name: "Palatino-Italic", size: 34)
    ?? NSFont(name: "Palatino", size: 34)
    ?? NSFont.systemFont(ofSize: 34, weight: .light)
```

Test that the font loaded: the resulting text should have visible serifs and lean slightly to the right. If it still looks like bold SF Pro, the font name is wrong. Check `NSFontManager.shared.availableFontFamilies` to find the exact name the system reports for Palatino.

### 2b — Stat numbers: Palatino roman

The numbers at the bottom of the home screen (3 / 0 / 86 in the screenshot, or equivalent) must use Palatino roman:

```swift
Text("\(statValue)")
    .font(.custom("Palatino", size: 28))
```

Labels beneath ("Check-ins this week", "Sources added", "Snapshots saved") stay in system UI at `text/secondary` (`#5A5854`).

```
- [ ] 2a. Greeting renders in Palatino Italic, ~34pt. Serifs are visible, text leans right.
- [ ] 2b. Stat numbers render in Palatino roman, ~28pt.
- [ ] 2b. Stat labels remain in system UI at text/secondary.
```

## (3) Home screen — button and tokens

### 3a — Primary action button (two states)

The check-in text area has a Save button (or equivalent CTA). It must have two states matching the same pattern as Seed's Plant button:

**Empty state** (nothing typed in the check-in field):
- Background: `prov/tint-surface` (`#E4F2EE`)
- Text: `prov/accent` (`#1D9E75`)
- Visible but subdued

**Active state** (user has typed something):
- Background: `prov/accent` (`#1D9E75`) solid
- Text: `#FAF6EF` (cream)
- Clear primary action

### 3b — Eyebrow label

"PROJECTS" (or equivalent section eyebrow above the project cards) must use `Brand.textMuted` (`#8C8A84`) — not a system grey.

### 3c — Status dots

Project status dots (the green circles next to project names in sidebar and home screen):
- Active/healthy: `prov/accent` (`#1D9E75`)
- Inactive: `Brand.textMuted` (`#8C8A84`)

### 3d — Sidebar selected row

Selected row background: `prov/tint-surface` (`#E4F2EE`). 0.5pt left stroke: `prov/tint-border` (`#B0D8CC`). No system grey. Applies in ALL views, not just home.

```swift
.background(isSelected ? Color("prov/tint-surface") : Color.clear)
.overlay(alignment: .leading) {
    if isSelected {
        Rectangle()
            .fill(Color("prov/tint-border"))
            .frame(width: 0.5)
    }
}
```

```
- [ ] 3a. Save/CTA button: tint-surface bg + accent text when empty; solid accent bg + cream when active.
- [ ] 3b. Eyebrow label uses Brand.textMuted.
- [ ] 3c. Status dots use prov/accent (active) and Brand.textMuted (inactive).
- [ ] 3d. Sidebar selected row: prov/tint-surface bg + 0.5pt prov/tint-border stroke. No system grey. Applies all views.
```

## (4) Project view — typography

**The project detail view (the screen showing "The Mythos of Aria" or "Procestra Dev") needs serif typography in several places.**

### 4a — Project display name

The large heading showing the project name ("The Mythos of Aria", "Procestra Dev") must use New York:

```swift
Text(project.displayName)
    .font(.custom("New York", size: 22))
    .fontWeight(.regular)
// Fallback: .font(.system(size: 22, design: .serif))
```

### 4b — Activity stat numbers

The numbers inside activity summary cards (check-in count, snapshot count, source count — shown as "13", "2", "0", "74" in the screenshot) must use Palatino roman:

```swift
Text("\(count)")
    .font(.custom("Palatino", size: 24))
```

Labels beneath ("versions", "check-ins", "snaps", "sources") stay in system UI at `text/secondary` (`#5A5854`).

### 4c — Inactive tab text color

The tab bar showing Overview / Sources / Artifacts / Versions (or equivalent) — inactive tabs must use `text/secondary` (`#5A5854`), not near-black. Active tab in `prov/accent` (`#1D9E75`) is correct; keep it.

```swift
// Inactive tab
Text(tabLabel)
    .foregroundColor(isActive ? Color("prov/accent") : Color(hex: "#5A5854"))
```

### 4d — Section headings

Section headings within the project overview ("Project Context", "Manuscripts", "Recent Activity", or equivalents) must use New York:

```swift
Text(sectionTitle)
    .font(.custom("New York", size: 15))
    .fontWeight(.medium)
```

```
- [ ] 4a. Project display name uses New York ~22pt.
- [ ] 4b. Activity stat numbers use Palatino roman ~24pt. Labels stay in system UI.
- [ ] 4c. Inactive tab labels use text/secondary (#5A5854).
- [ ] 4d. Section headings use New York ~15pt medium.
```
