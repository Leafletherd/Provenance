# PR-24 — Project view header continuity + tab-chip restyle

The Home view is right. The project view still has the old header band. Bring the project view in line and restyle its tabs to match the filter chip design.

---

## (0) Read first

1. Open the installed Provenance Home view. Note: the cream/tan background extends fully from the bottom of the window up under the title bar. No seam. This is the target for the project view too.
2. Open the project view (click any project in the sidebar). Note: there is a band of slightly different color at the top, above the tab bar. This is the OLD header that shouldn't be there.
3. Find the project view root (the view that renders Overview/Sources/Artifacts/etc. with the tab bar at top). Look for any explicit toolbar background, header strip, or NavigationStack layout that creates the band.
4. Find the filter chip code for the Artifacts tab's "All / Seeds / Notes / Images / Audio / Other" pills. Note their exact styling — corner radius, padding, selected vs unselected backgrounds, text colors.

---

## (1) Item A — Continuous background in project view

**The bug:** in the project view, the area above the tab bar (where the toolbar/title bar sits) shows a band of different color from the main content area. This is the "old header" the user is calling out. The Home view doesn't have this — its background flows under the title bar seamlessly.

### A1 — Apply Home view's pattern to project view

Identify what makes the Home view background continuous. It likely uses:
- `.ignoresSafeArea()` on the root background color
- `.toolbarBackground(Brand.surfaceBase, for: .windowToolbar)` applied at the Scene root (per PR-22 B1)
- No additional VStack with a "header" area inserted above the content

The project view must use the same approach:
- Root background: `Brand.surfaceBase` (or whichever token Home uses) with `.ignoresSafeArea()`
- Tab bar floats over this background — it does not have its own opaque background that creates a visible band
- No `VStack { HeaderView(); ContentView() }` pattern with a separate header strip

### A2 — Keep the tab-bar divider

The tab bar can still have a divider line at its bottom edge (a thin separator between the tabs and the content below). That's expected and desired. What must go is the BAND above the tabs (the old header strip).

```swift
// Tab bar:
HStack { /* tabs */ }
    .padding(.horizontal, 16)
    .padding(.vertical, 8)
    // NO opaque background — just the bottom divider:
    .overlay(alignment: .bottom) {
        Rectangle()
            .fill(Brand.borderSubtle)
            .frame(height: 0.5)
    }
```

The tab bar's container does NOT have a `.background(...)` that creates a visible band. It sits transparently over the continuous `surfaceBase` background.

### A3 — Verify

Switch between Home and a project view. Both must show the same continuous background flowing from the bottom of the window up under the title bar. No visible seam or band at the top of the project view.

```
- [ ] A1. Project view root uses the same background pattern as Home view.
- [ ] A2. Tab bar has no opaque background. Bottom divider preserved.
- [ ] A3. Visual confirmation: Home and project view backgrounds look continuous (same color all the way up).
```

## (2) Item B — Project tabs adopt filter-chip pill design

**The reference:** the "All / Seeds / Notes / Images / Audio / Other" filter chips on the Artifacts tab look like rounded pills with the selected one having a filled background. The project view tabs (Overview / Sources / Artifacts / Versions / Check-ins / Ledger / Export) should use the SAME design.

### B1 — Read the filter chip styling

Find the filter chip view code. Note its exact properties:
- Corner radius (likely 8pt or fully rounded)
- Horizontal padding (likely 12pt)
- Vertical padding (likely 6pt)
- Selected background color (light + dark mode)
- Selected text color (light + dark)
- Unselected text color
- Font size and weight
- Spacing between chips

### B2 — Apply to project view tab bar

Restyle each tab in the project view tab bar to match the filter chip:

```swift
ForEach(ProjectTab.allCases, id: \.self) { tab in
    Button(action: { selectedTab = tab }) {
        Text(tab.label)
            .font(/* matches filter chip font */)
            .foregroundColor(
                selectedTab == tab
                    ? /* chip selected text color */
                    : Color(Brand.textSecondary)
            )
            .padding(.horizontal, /* matches chip */)
            .padding(.vertical, /* matches chip */)
            .background(
                selectedTab == tab
                    ? RoundedRectangle(cornerRadius: /* matches chip */)
                        .fill(/* chip selected bg color */)
                    : nil
            )
    }
    .buttonStyle(.plain)
}
```

### B3 — Remove the underline

The previous tab design used a 2pt `prov/accent` underline for the selected tab. With the chip pill design, that underline is no longer needed — the filled pill background is the selection indicator. **Remove the underline.**

### B4 — Both modes

The chip styling already adapts via brand tokens. Verify both light and dark mode look correct:
- Light: tan-on-tan selected pill, with selected text in textPrimary
- Dark: matching darker selection backing

If the existing filter chip's selected color doesn't read well at the project-tab scale, use the same exact color (so the two stay visually unified). Don't fork a new shade.

```

- [ ] B3. 2pt prov/accent underline removed from project tabs.
- [ ] B4. Both light and dark mode tested.
```
