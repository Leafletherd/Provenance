# PR-16 — Provenance surface and tab consistency

User testing 2026-05-16 confirms Provenance has visual inconsistency between tabs in both appearance modes — Home/Export use one treatment in light mode, the other tabs use another in dark mode. The user has identified the **canonical** treatment from each mode and wants it applied to all tabs.

This prompt also covers two specific polish items (toolbar pill extension, artifact card dark bg) and removes the Recent Exports section per user direction.

## The canonical layered surface pattern

Provenance uses three layered surfaces. In the canonical (correct) state:

**Light mode:**
- Toolbar (top): white / `Brand.surfaceRaised` background
- Sidebar (left): white-ish color (a slight off-white that stands out), with a smooth vertical gradient from the toolbar's white at the top into the main body color toward the bottom
- Main body (right pane): tan / `Brand.surfaceBase`
- Effect: the toolbar and sidebar "frame" the tan body

**Dark mode:**
- Toolbar: a slightly darker variant than the body — `Brand.surfaceRaised` (dark)
- Sidebar: same as toolbar, with the same vertical gradient into the body
- Main body: warm dark / `Brand.surfaceBase` (dark) — the user reports this reads as "blueish-grey"
- Effect: same layered framing, dark variant

The pattern is consistent: **toolbar and sidebar are visually one piece (the chrome); the body sits inside the chrome with a contrasting surface.**

## Current state of the bug

- **Light mode:** Only Home tab and Export tab follow the pattern correctly. The other tabs (Overview, Sources, Artifacts, Versions, Check-ins, Ledger) have a plain white main-body background — losing the tan/chrome contrast.
- **Dark mode:** Home tab and Export tab have a *different* background from the other tabs. The other tabs (Overview etc.) use the correct dark body color (what the user called blueish-grey). Settings is also correct. Home and Export are the ones to fix.

## Reference: the canonical implementations

- **Light mode reference:** Home tab and Export tab as they currently render in light. Use these as the source of truth for the OTHER tabs to match.
- **Dark mode reference:** Overview tab (and Sources, Artifacts, Versions, Check-ins, Ledger, Settings) as they currently render in dark. Use these as the source of truth for Home and Export to match.

If `Brand.surfaceBase` and related tokens are already dynamic (per REV-3 / SB-27 patterns), this is largely about ensuring every tab view uses Brand tokens consistently, not custom backgrounds.

## (1) Apply canonical surface pattern to all tabs

Audit each project tab view in `Sources/Views/Project/`:
- `OverviewView.swift`
- `SourcesView.swift`
- `ArtifactsView.swift`
- `VersionsView.swift`
- `CheckInsView.swift`
- `LedgerView.swift`
- `ExportView.swift`

And Home view in `Sources/Views/Home/HomeView.swift`.

For each view's main container:

```swift
.background(Brand.surfaceBase)   // tan in light, warm-dark in dark
```

NOT `.background(Color.white)` or any other literal. The tab content area renders directly on `Brand.surfaceBase`. Cards, panels, and inner widgets sit on `Brand.surfaceRaised` or other contrasting tokens — but the tab's main background is `Brand.surfaceBase`.

If any tab is currently using a different background (`Brand.surfaceRaised`, `Color.white`, a hardcoded color), change it to `Brand.surfaceBase`.

## (2) Sidebar gradient

The sidebar visually transitions from the toolbar's color at the top to the body's color at the bottom — a vertical gradient creating the impression that the toolbar and sidebar are one continuous surface that frames the body.

Verify the sidebar renders with this gradient in BOTH modes. The gradient:
- Light: from `Brand.surfaceRaised` (white) at top → smoothly into `Brand.surfaceSunken` (tan-100) at bottom (or `Brand.surfaceBase` — pick whichever matches the existing Home tab's correct rendering)
- Dark: from `Brand.surfaceRaised` (warm-dark) at top → into `Brand.surfaceBase` (the body color) at bottom

If the gradient is missing or rendering with hardcoded colors, fix to use dynamic `Brand` tokens with a `LinearGradient`.

## (3) Toolbar pill extension

The pill-shaped toolbar group containing the Home icon and "+ Connect Project" button is currently too crowded — the icons sit near the edges of the pill with little visual breathing room.

Fix:
- Increase the pill's horizontal padding so the Home icon and the + button have ~8pt of breathing room on each side
- Don't make the pill significantly wider than needed — just enough that the icons aren't crowded
- The pill's background and border stay the same (just the internal padding grows)

This is a tiny adjustment. Look at the current rendering, find the toolbar group container, increase the `.padding(.horizontal, N)` value (or whatever padding modifier is in use).

## (4) Artifact card dark mode background

In the Artifacts tab, the cards rendering individual artifacts currently use a different background color in dark mode than the cards in the Check-ins tab.

Fix:
- Find the artifact card view (probably `Sources/Views/Project/ArtifactsView.swift` or a helper view)
- Find the check-in entry card view (`CheckInsView.swift` or similar)
- Identify the background color the check-in card uses in dark mode (it'll be a `Brand.*` token or a dynamic color)
- Apply the same background to the artifact card

Both card types should look visually identical in dark mode — same background, same border treatment, same elevation feel.

## (5) Remove Recent Exports section

In the Export tab (per PR-7), remove the "Recent exports" section entirely. Per user direction.

This was originally added so users could find files they'd exported recently. Removing it keeps the Export tab focused on the action of exporting.

- Delete the section's view code from `ExportView.swift`
- Delete any supporting state (recent-exports tracking, if any)
- Don't delete the actual exported files on disk — only the in-app UI listing them
