# PR-29 — Sidebar tan only; right column is cream top-to-bottom. Home view untouched.

PR-28 painted a tan band across the full window top. That was wrong. Look at Seed once more:

- The sidebar is tan and extends UP under the title bar — but ONLY on the left side.
- The right side of the window (where the pill/header sits AND where the content sits) is **cream from top to bottom**.
- The vertical line where sidebar tan meets right-column cream is the only color change.

And:
- In the top zone (pill/header area): the vertical seam is a HARD edge.
- In the body zone (below the header): the vertical seam is a SOFT gradient (~20–30pt).

The home view is unrelated — it was correct before PR-28 (single continuous cream). Restore it.

## (1) Item A — Revert the full-width tan band

### A1 — Color zones

```
┌─────────────┬────────────────────────────────────────────────┐
│             │                                                │
│  TAN        │  CREAM (header area, pill sits here)           │
│  SIDEBAR    │                                                │
│  (also     │░ ← hard edge at top of seam                     │
│   extends   │░░                                              │
│   under     │░░░ ← gradient blend in body zone               │
│   title    │░░                                               │
│   bar)      │░  CREAM (content)                              │
│             │                                                │
└─────────────┴────────────────────────────────────────────────┘
```

- **Sidebar:** tan (`Brand.surfaceSidebar`), extends UP under the title bar on the LEFT only. Does NOT extend across the full window top.
- **Right column (entire vertical extent):** cream (`Brand.surfaceBase`) — from top to bottom. The pill at the top right sits over cream. The tab bar sits over cream. The content sits over cream. All the same cream, no internal color change.
- **Vertical seam between sidebar tan and right cream:**
  - In the top zone (~44pt, the height of the title bar / pill row): HARD edge — abrupt vertical color change with no blending.
  - In the body zone (below the top zone): SOFT gradient — 20–30pt wide, tan fading into cream.

### A2 — Remove the full-width tan band

Whatever PR-28 did to paint the whole top tan: undo it. Specifically:

- Remove any `window.backgroundColor = NSColor(Brand.surfaceSidebar)` set at the window level (or change it to cream — see A3).
- Remove any `.toolbarBackground(Brand.surfaceSidebar, for: .windowToolbar)` that applies tan to the toolbar across the full width.
- Remove the "Layer 1: full window tan background" pattern from PR-28's implementation.

### A3 — Re-apply the correct pattern

Window background goes back to cream (or transparent so SwiftUI views control it):
```swift
window.backgroundColor = NSColor(Brand.surfaceBase)
window.titlebarSeparatorStyle = .none
window.toolbar?.showsBaselineSeparator = false
```

Toolbar background goes back to cream:
```swift
.toolbarBackground(Brand.surfaceBase, for: .windowToolbar)
.toolbarBackground(.visible, for: .windowToolbar)
```

The sidebar paints its own tan background that extends under the title bar on its OWN side only (PR-23's `Brand.surfaceSidebar.ignoresSafeArea()` pattern, which was correct):

```swift
// In the sidebar root:
ZStack(alignment: .top) {
    Brand.surfaceSidebar
        .ignoresSafeArea()       // extends under title bar — on the sidebar's column only
    SidebarMaterialKiller()
        .frame(width: 0, height: 0)
    VStack(spacing: 0) {
        // sidebar content
    }
}
```

The right column does NOT paint a separate background — it inherits the cream window background. Only the body area's leading edge gets a gradient overlay:

```swift
// In the right column's body container (below the toolbar/pill area):
ZStack(alignment: .leading) {
    LinearGradient(
        colors: [Brand.surfaceSidebar, Brand.surfaceBase],
        startPoint: .leading,
        endPoint: .trailing
    )
    .frame(width: 24)
    .frame(maxWidth: .infinity, alignment: .leading)
    
    // Tab bar + content
    VStack(spacing: 0) {
        TabBarView()
        ScrollView { ContentView() }
    }
}
```

The header zone (the toolbar / pill row at top of the right column) does NOT have the gradient overlay — only the body does. That preserves the hard edge at the top of the seam.
