# PR-28 — Header band is tan (full-width), not cream. Fix sidebar-toggle state flip.

PR-27 said the header band should be cream. That was wrong. Looking at Seed's bench more carefully:

- The header band is **tan** — same color as the sidebar — spanning the FULL width of the window.
- The content area BELOW the header band (right of sidebar) is cream.
- The horizontal color change between tan-header and cream-content is the visible "dividing line."
- The vertical seam between sidebar-tan and content-cream (below the header band) is where the gradient lives.

Also: toggling the sidebar in the current Provenance build switches between two different header appearances — neither correct. That state flip needs to go.

---

## (0) Read first

1. Open Seed's bench view side-by-side with Provenance's project view. Trace exactly what color is where:
   - **Top band (under title bar, ~40–48pt tall):** tan (`Brand.surfaceSidebar`) across the FULL window width — over the sidebar, AND over the right column header area.
   - **Below the top band, left:** sidebar tan continues.
   - **Below the top band, right:** content area is cream (`Brand.surfaceBase`).
   - **Boundary between top band and content area (horizontal):** abrupt color change — the visible "divider."
   - **Boundary between sidebar and content area (vertical, below the top band only):** soft horizontal gradient (tan → cream over ~20–30pt).

2. In Provenance, find why toggling the sidebar produces two different header appearances. The likely cause: the toolbar background is being applied conditionally based on a state that changes when the sidebar toggles (e.g., a `@State var` recomputes, or the `NavigationSplitView` re-lays out and a previously-suppressed default toolbar tint comes through).

## (1) Item A — Correct the header band color: tan, full width

### A1 — The required structure

```
┌──────────────────────────────────────────────────────────────┐
│                  TAN HEADER BAND (full width)                │   ← surfaceSidebar
├─────────────┬────────────────────────────────────────────────┤
│             │                                                │
│  TAN        │  CREAM CONTENT                                 │   ← surfaceBase
│  SIDEBAR    │  (with gradient seam at left edge)             │
│             │                                                │
└─────────────┴────────────────────────────────────────────────┘
```

- Top band: `Brand.surfaceSidebar` (tan), spanning the full window width, extending under the title bar.
- Sidebar (below top band, left): `Brand.surfaceSidebar` tan, continuous with the top band.
- Right column content (below top band): `Brand.surfaceBase` cream.
- Horizontal boundary (between top band and content): abrupt tan→cream color change. NO line, just the color change.
- Vertical boundary (between sidebar tan and content cream, below the top band): soft horizontal gradient over ~20–30pt.

### A2 — Implementation approach

The simplest way to achieve this: paint the entire window background tan, then overlay a cream rectangle in the bottom-right (content area) with a left-edge gradient.

```swift
ZStack {
    // Layer 1: full window tan background
    Brand.surfaceSidebar
        .ignoresSafeArea()
    
    // Layer 2: cream content rectangle, positioned in the lower-right
    HStack(spacing: 0) {
        // Spacer for sidebar width
        Color.clear.frame(width: sidebarWidth)
        
        VStack(spacing: 0) {
            // Spacer for header band height
            Color.clear.frame(height: headerBandHeight)  // ~44pt
            
            // Cream content area with gradient at leading edge
            ZStack(alignment: .leading) {
                Brand.surfaceBase
                
                LinearGradient(
                    colors: [Brand.surfaceSidebar, Brand.surfaceBase],
                    startPoint: .leading,
                    endPoint: .trailing
                )
                .frame(width: 24)
                .frame(maxWidth: .infinity, alignment: .leading)
                
                // Actual content (tab bar, tiles, etc.)
                VStack(spacing: 0) {
                    TabBarView()
                    ScrollView { ContentView() }
                }
            }
        }
    }
    
    // Layer 3: sidebar content and right-column header content sit on top
    HSplitView { /* sidebar | right column with content */ }
}
```

Or — if `NavigationSplitView` is used and changing this is invasive — paint the window background tan and only paint the content area cream as a child view of the right detail column:

```swift
// At window/scene root:
window.backgroundColor = NSColor(Brand.surfaceSidebar)

// In the detail column:
VStack(spacing: 0) {
    Color.clear.frame(height: 44)  // reserves header band space — stays tan from window bg
    
    ZStack(alignment: .leading) {
        Brand.surfaceBase
        LinearGradient(
            colors: [Brand.surfaceSidebar, Brand.surfaceBase],
            startPoint: .leading,
            endPoint: .trailing
        )
        .frame(width: 24)
        .frame(maxWidth: .infinity, alignment: .leading)
        
        VStack(spacing: 0) {
            TabBarView()
            ScrollView { ContentView() }
        }
    }
}
```

Either approach is acceptable. Pick whichever fits the existing architecture more cleanly. Verify the result visually against Seed's bench — they should look like siblings.

## (2) Item B — Fix sidebar-toggle state flip

**The bug:** toggling the sidebar in the current build switches the header appearance between two different (both wrong) styles.

### B1 — Diagnose

Find why this happens. Likely causes:
- A `@State var` recomputes when `NavigationSplitView` collapses/expands the sidebar, and the toolbar background is conditionally applied based on that state.
- The `.toolbarBackground` modifier is on a view that mounts/unmounts when the sidebar visibility changes.
- The window's `backgroundColor` is being set in one mode but not the other.

### B2 — Fix

Apply the toolbar background and window backgroundColor at the App/Scene root level (not on a view that re-mounts with sidebar toggles). The toolbar background must remain the same regardless of sidebar visibility.

```swift
@main
struct ProvenanceApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
                .toolbarBackground(Brand.surfaceSidebar, for: .windowToolbar)
                .toolbarBackground(.visible, for: .windowToolbar)
                .background(WindowConfigurator())  // configures NSWindow
        }
    }
}

private struct WindowConfigurator: NSViewRepresentable {
    func makeNSView(context: Context) -> NSView {
        let v = NSView()
        DispatchQueue.main.async {
            if let w = v.window {
                w.backgroundColor = NSColor(Brand.surfaceSidebar)
                w.titlebarSeparatorStyle = .none
                w.toolbar?.showsBaselineSeparator = false
            }
        }
        return v
    }
    func updateNSView(_ nsView: NSView, context: Context) {}
}
```

### B3 — Verify

In the installed build:
1. Open Provenance. Note the header.
2. Toggle the sidebar (click the sidebar toggle in the toolbar). Note the header.
3. Toggle again. Note the header.
4. All three states must have the IDENTICAL tan header band. No flicker. No color shift.
