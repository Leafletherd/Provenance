# PR-30 — Three-tone chrome (sidebar tan / header darker-cream / body lighter-cream) + gradient artifact fix

PR-29 made the right column one uniform cream. Seed actually has a subtle but visible color shift between the header zone and the body zone — both in the right column. The header is **slightly darker** than the body. That subtle difference IS the visual divider the user has been pointing at all along.

Three colors. Three zones. Two seams. Get it right this time.

---

## (0) Read first

1. Open the installed Seed app, Bench view. Hold the user's screenshot beside it. Look at the right column:
   - The top portion (where the pill sits, where "Bench 2 +" is) has a SLIGHTLY DARKER cream tone than the body below.
   - It's subtle. Not a tan band. Not white. Just a perceptible shade darker than the body cream.
2. Check `BrandTokens.swift` for these tokens:
   - `Brand.surfaceSidebar` — sidebar tan
   - `Brand.surfaceBase` — main cream (the body color)
   - `Brand.surfaceSunken` — a slightly darker cream (used for stat tiles per PR-22's C4)
   - `Brand.surfaceFloating` — REV-7 floating pill color
3. Verify which token Seed uses for its right-column header zone. If `surfaceSunken` matches visually, use it. If not, create a new `surfaceHeader` token between `surfaceSidebar` and `surfaceBase`.

## (1) Item A — Three color zones

### A1 — The layout

```
┌─────────────┬────────────────────────────────────────────────┐
│             │  RIGHT COLUMN HEADER ZONE (darker cream)       │
│  TAN        │  ░░  ← gradient blend in header zone           │
│  SIDEBAR    │  (pill, tabs, project header content)          │
│  (extends   │┄┄┄┄ ← subtle color shift, no line ┄┄┄┄┄┄┄┄┄┄┄┄ │
│   under     │  ░░░  ← gradient blend in body zone            │
│   title)    │  RIGHT COLUMN BODY (lighter cream)             │
│             │  (project content, tiles)                      │
└─────────────┴────────────────────────────────────────────────┘
```

- **Sidebar:** `Brand.surfaceSidebar` (tan). Extends up under title bar on its column only.
- **Right column header zone:** ~44pt tall at the top of the right column (where the toolbar / pill / project name sit, and below that where the tab bar sits if it lives in the header area). Color: `Brand.surfaceSunken` (slightly darker cream than the body). If `surfaceSunken` is too dark, create a new `Brand.surfaceHeader` token tuned to match Seed exactly.
- **Right column body zone:** below the tab bar (or wherever the header zone ends). Color: `Brand.surfaceBase` (cream).
- **Horizontal divider between header zone and body zone:** subtle color shift only. No drawn line. The slight tone difference does the work.

### A2 — Determine where the header zone ends

The horizontal boundary between header zone (darker cream) and body zone (lighter cream) sits **right below the tab bar**. So:
- Top ~44pt (toolbar/pill area): surfaceSunken
- Tab bar row (Overview / Sources / Artifacts / etc.): surfaceSunken (still in the header zone)
- Below the tab bar bottom edge: surfaceBase

OR — alternative if Seed's pattern is different — the boundary may be at the very top of the tab bar (so the tab bar itself sits in the body zone). Pick whichever matches Seed visually. Confirm in the running Seed app before committing.

### A3 — Vertical gradient at sidebar/right-column seam

In BOTH zones (header and body), apply a soft horizontal gradient blending the sidebar tan into the right-column color of that zone:

- **In header zone:** gradient from `Brand.surfaceSidebar` → `Brand.surfaceSunken` (the header cream). 20–30pt wide. Anchored to the leading edge of the right column's header.
- **In body zone:** gradient from `Brand.surfaceSidebar` → `Brand.surfaceBase` (the body cream). 20–30pt wide. Anchored to the leading edge of the right column's body.

The two gradient strips align vertically so the user sees ONE continuous soft seam from top to bottom of the window — but with the seam's right-side color shifting subtly at the horizontal boundary.

### A4 — Implementation

```swift
// Right column structure:
VStack(spacing: 0) {
    // HEADER ZONE
    ZStack(alignment: .leading) {
        Brand.surfaceSunken  // base for header zone
        
        // Vertical gradient on leading edge (in header zone only)
        LinearGradient(
            colors: [Brand.surfaceSidebar, Brand.surfaceSunken],
            startPoint: .leading,
            endPoint: .trailing
        )
        .frame(width: 24)
        .frame(maxWidth: .infinity, alignment: .leading)
        
        // Header content (pill area + tab bar)
        VStack(spacing: 0) {
            // toolbar/pill (built by SwiftUI .toolbar)
            TabBarView()
        }
    }
    
    // BODY ZONE
    ZStack(alignment: .leading) {
        Brand.surfaceBase  // base for body zone
        
        // Vertical gradient on leading edge (in body zone only)
        LinearGradient(
            colors: [Brand.surfaceSidebar, Brand.surfaceBase],
            startPoint: .leading,
            endPoint: .trailing
        )
        .frame(width: 24)
        .frame(maxWidth: .infinity, alignment: .leading)
        
        // Body content
        ScrollView { ContentView() }
    }
}
```

Each zone has its own surface color AND its own gradient that blends from sidebar-tan into that zone's surface color. The two gradients line up vertically.

- [ ] A1. Sidebar = surfaceSidebar (tan).
- [ ] A1. Right column header zone = surfaceSunken (or new surfaceHeader). Slightly darker cream than body.
- [ ] A1. Right column body zone = surfaceBase. Lighter cream.
- [ ] A2. Horizontal boundary between zones = subtle color shift, no drawn line. Boundary position matches Seed (just below tab bar or at top of tab bar — pick what matches).
- [ ] A3. Gradient at sidebar seam in BOTH zones — each gradient blends sidebar tan into that zone's specific surface color.

## (2) Item B — Fix the weird gradient artifact behind the Overview tab

**The bug:** the close-up screenshot the user sent shows a diagonal/blob-shaped gradient visible behind the Overview tab pill. This shouldn't be there.

### B1 — Diagnose

This gradient artifact is likely:
- The body-zone gradient overlay extending into the header zone (because it's not bounded by its container's frame).
- A SwiftUI `LinearGradient` with the wrong `startPoint`/`endPoint` producing an unintended diagonal.
- A `.background` modifier with a gradient that's not properly clipped.
- The gradient at the sidebar seam bleeding right because its width isn't constrained.

### B2 — Fix

After Item A's restructuring, the gradient lives strictly inside each zone's ZStack — clipped to that zone's frame. Verify:

```swift
ZStack(alignment: .leading) {
    Brand.surfaceBase
    LinearGradient(...)
        .frame(width: 24)
        .frame(maxWidth: .infinity, alignment: .leading)
    // ...content...
}
.clipped()  // ensure nothing bleeds out
```

The `.clipped()` at the end of each zone's ZStack ensures the gradient and any other overlay stays inside that zone. No diagonal bleed, no blob.

### B3 — Verify

Open the Overview tab. The area behind the tab pills (Overview / Sources / Artifacts / etc.) must be a flat color — no visible gradient. The gradient is ONLY at the leading edge (sidebar boundary), spanning a ~24pt strip.

```
- [ ] B1. Identified what was producing the diagonal artifact.
- [ ] B2. Each zone's gradient is clipped to its own frame.
- [ ] B3. Visible gradient appears ONLY at the leading edge sidebar seam. Nowhere else.
```
