## (2) Item B — White header regression (diagnostic clue this time)

**The new clue:** closing and reopening the sidebar makes the white header disappear. This means the toolbar background IS set correctly, but it's not applied on initial layout — only after a layout invalidation triggered by the sidebar toggle.

**Diagnosis:** the `.toolbarBackground` modifier is conditional on the view's mounted state, OR it's applied to a sub-view that's not in the responder chain on first launch.

### B1 — Force the toolbar background on initial mount

Apply `.toolbarBackground` at the OUTERMOST level — on the root `App` or `Scene`, not on individual views:

```swift
@main
struct ProvenanceApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
                .toolbarBackground(Color("surface/base"), for: .windowToolbar)
                .toolbarBackground(.visible, for: .windowToolbar)
                .toolbar(.visible, for: .windowToolbar)
        }
    }
}
```

### B2 — Also at the NSWindow level

In the AppDelegate or wherever NSWindow is configured:
```swift
window.backgroundColor = NSColor(named: "surface/base")
window.titlebarSeparatorStyle = .none
window.toolbar?.showsBaselineSeparator = false
```

### B3 — Force a layout pass on appear

If the issue is a missing layout pass on first mount, force one:
```swift
.onAppear {
    DispatchQueue.main.async {
        NSApp.keyWindow?.toolbar?.validateVisibleItems()
        NSApp.keyWindow?.layoutIfNeeded()
    }
}
```

### B4 — Verify

1. Launch Provenance fresh (quit completely first).
2. Open the app. The header MUST be cream `surface/base`, NOT white.
3. Do not need to toggle the sidebar to see it correctly.

```
- [ ] B1. Toolbar background applied at App/Scene root level.
- [ ] B2. NSWindow-level configuration applied.
- [ ] B4. Fresh launch shows cream header immediately. No white flash, no sidebar-toggle needed.
```
